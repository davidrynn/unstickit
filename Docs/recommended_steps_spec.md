# Unstuck — Recommended Steps Spec

## Status
**Draft** — additive post-MVP feature

**Sequencing:** Build this before **Come back tomorrow**. Recommended Steps is the local storage foundation that deferred tomorrow steps use.

---

## 1. Purpose

Recommended steps give users a small, temporary place to keep track of activation steps generated by Unstuck.

The feature should answer:

> "What small actions has the app suggested recently, and what do I want to do with them?"

It is not a full task manager, project tracker, or permanent archive.

---

## 2. Product Intent

Unstuck produces one concrete next step at a time. Some of those steps will be useful later, but making every step permanent would move the app toward task management.

Recommended steps should therefore be:

- temporary by default
- easy to dismiss
- easy to save intentionally
- lightweight enough to review with swipes
- focused on movement, not completion systems

Default rule:

> A recommended step is deleted after 7 days unless the user saves it.

---

## 3. User Experience

### Entry Point

Add a quiet entry point from S1:

**Recent steps**

Recommended placement:

- small toolbar button or text button near the top of S1
- show only when at least one recommended step exists
- do not make this the primary app surface

### Main View

Screen title:

**Recent steps**

Empty state:

**No recent steps**

Body:

**Steps you keep for later will show up here for a week.**

### Step Card Content

Each card shows:

- step text
- source date for recent unsaved steps, e.g. **Today**, **Yesterday**, or **May 20**
- saved indicator if saved

For saved steps older than 7 days, suppress the date in the default card view. The date can appear in a detail view later, but the main list should not feel like an archive.

Do not show the original brain dump by default. It may contain sensitive or noisy text.

---

## 4. Swipe Model

Use a card stack or vertical list with horizontal swipe actions.

Recommended gestures:

- Swipe right: **Start**
- Swipe left: **Let go**
- Long press or overflow menu: **Save**

Accessible button equivalents must be available for every swipe action.

### Start

Meaning:

The user is choosing to act on this step now.

Behavior:

- leave the step in the list until it expires, is saved, or is dismissed
- return to S1 fresh
- do not create a completion streak or history requirement

Button label:

**Start**

### Let Go

Meaning:

The user does not need this step anymore.

Behavior:

- delete the step immediately
- no confirmation required for unsaved steps
- require confirmation before deleting saved steps

Button label:

**Let go**

Confirmation for saved step:

**Delete saved step?**

Body:

**This removes it from your saved steps.**

Actions:

- **Delete**
- **Cancel**

### Save

Meaning:

The user wants to keep this step beyond the 7-day window.

Behavior:

- set `isSaved = true`
- remove `expiresAt` or ignore expiration while saved
- keep it available in the list

Button label:

**Save**

Saved state label:

**Saved**

Unsave action:

**Unsave**

Behavior:

- set `isSaved = false`
- set `expiresAt` to 7 days after the original `createdAt`
- if that date is already in the past, delete the step immediately after confirmation

---

## 5. Data Model

```swift
enum RecommendedStepSource: String, Codable {
    case nextStep
    case fallbackStep
    case deferredTomorrow
}

enum RecommendedStepStatus: String, Codable {
    case active
    case dismissed
}

struct RecommendedStep: Codable, Identifiable {
    let id: UUID
    let text: String
    let fallbackText: String?
    let source: RecommendedStepSource
    let originalBrainDump: String?
    let createdAt: Date
    var availableOn: Date?
    var expiresAt: Date?
    var status: RecommendedStepStatus
    var isSaved: Bool
}
```

Storage:

- local-only
- on-device
- user-deletable
- no account or sync requirement

Out of scope: sync. If the user reinstalls the app or switches devices, recent steps are not preserved.

Retention:

- unsaved steps expire 7 days after `createdAt`
- saved steps do not expire
- tapping **Start** does not change retention; the step remains recoverable until it expires, is saved, or is dismissed
- dismissed steps are deleted immediately

---

## 6. Creation Rules

Create a recommended step only when the user expresses intent to keep, revisit, or continue from it.

Create one when:

- the user taps **Save this step**
- the user taps **Come back tomorrow**
- the user taps **I'm still stuck** and chooses to keep the fallback step

Do not create duplicate active steps with identical text generated within the same session.

Do not automatically persist every generated `nextStep`. A generated step only enters **Recent steps** after explicit user intent.

For MVP-plus simplicity, store the generated `nextStep` as `text`. Store the `fallbackStep` as `fallbackText` only when the user explicitly saves, defers, or keeps it.

---

## 7. Relationship to Come Back Tomorrow

The **Come back tomorrow** feature creates exactly one recommended step with:

```swift
source = .deferredTomorrow
text = nextStep
fallbackText = fallbackStep
availableOn = next local calendar day at 5:00 AM, but never sooner than 6 hours after createdAt
isSaved = false
expiresAt = createdAt + 7 days
```

On the next day, the step appears in the return card and in **Recent steps**.

If the user lets go of or saves the step from either surface, the other surface should reflect the same state. Tapping **Start** does not remove the step; it remains recoverable until it expires or the user dismisses it.

---

## 8. AI Behavior

This feature does not require new AI generation.

It reuses:

- `nextStep`
- `fallbackStep`
- original brain dump metadata if already available

Do not ask the AI to generate task names, categories, priorities, or summaries for the list. That would push the feature toward project management and increase model dependency.

---

## 9. Copy Guidelines

Follow the shared copy rules in [copy_principles.md](copy_principles.md).

Use:

- **Recent steps**
- **Start**
- **Let go**
- **Save**
- **Saved**
- **Unsave**
- **This will disappear in 7 days unless saved.**

The language should keep the feature temporary and low-pressure.

---

## 10. Edge Cases

- If the list has only expired unsaved steps, delete them before rendering and show the empty state.
- If a saved step is unsaved after its original expiration date, ask whether to delete it now.
- If the user has more than 20 unsaved recent steps, keep only the 20 most recent unsaved steps and delete the rest. Saved steps are not included in this cap.
- If the user creates the same step text twice in one session, keep one item.
- If original brain dump text is stored, do not show it in the list without an explicit reveal action.

---

## 11. Acceptance Criteria

- [ ] Generated `nextStep` appears in **Recent steps** only after explicit save, defer, or keep intent
- [ ] Recent steps are stored locally only
- [ ] Unsaved steps auto-delete after 7 days
- [ ] Saved steps do not auto-delete
- [ ] User can swipe right or tap **Start** without removing the step from the list
- [ ] User can swipe left or tap **Let go**
- [ ] User can save, unsave, and delete saved steps
- [ ] Deferring a step creates exactly one `RecommendedStep` with `source == .deferredTomorrow`
- [ ] Swipe actions have accessible button equivalents
- [ ] Empty state appears when no active or saved steps exist
- [ ] The feature does not use task-manager language such as overdue, priority, complete, or to-do
